So the core idea was to train a 3 billion parameter language model to generate search queries that would improve product retrieval. The setup was inspired by DeepRetrieval — we used online RL where the model generates candidate queries, and then we evaluate those queries against the retrieval system to get a reward signal...

The reward itself was composite. We had retrieval recall as one component — did the generated query actually retrieve relevant products? And then we had the LLM judge, a 70B model that scored query quality on multiple dimensions: relevance, diversity, and specificity. We combined these into a single scalar reward...

For the optimization, we used PPO because we needed stable training. The reward signal from the LLM judge is inherently noisy — it's a large model making judgments, and there's variance in those judgments. PPO's clipped surrogate objective helps prevent the policy from making too-large updates based on noisy rewards...

Okay, so first I want to make sure I understand the scope. When we say trust and safety for Alexa+ LLM responses, are we talking about filtering the generated text before it reaches the user? And is this for all Alexa+ interactions or a specific domain like shopping or health?...

Assuming it's a general-purpose safety system, I'd approach this with a multi-layer architecture, similar to what I built at Walmart for product content safety. The key insight is that you want fast, cheap filters for obvious violations and then increasingly sophisticated evaluation for edge cases...

Layer one is rule-based: regex patterns, blocklists, known harmful content signatures. This catches maybe 80% of clearly bad content with sub-millisecond latency.
